# 모두의 대시보드 (demo-dashboard-drop)

> 표시명은 **"모두의 대시보드"** (v0.2, 마스터 피드백 2026-07-23). 저장소명·Vercel 프로젝트·URL(`itda-demo-dashboard-drop`)은 유지 — 표시명만 변경. 적용 지점: Header 로고·index.html(title/description/JSON-LD/og/twitter/apple-title)·vite manifest(name·short_name)·scripts(og·splash 텍스트)·문서. 형제 앱 series.js 의 name 도 순차 반영 대상(이 저장소 self 항목은 반영됨).

CSV 즉석 대시보드 PWA. 교육용 데모 시리즈 B축 — 원형은 교육 실습 과제 'Drop 매출 대시보드'(수강생 결과물 패턴 ④). "실습 한 조각 → 완성 제품" 도약을 보여주는 것이 목적.

## 배경과 목적

사용자 문제: 매출·실적 CSV를 훑어보고 싶은데, 클라우드 BI 도구에 회사 데이터를 올리는 것이 보안 규정상 막혀 있거나 부담스럽다.
해결 접근: CSV를 끌어다 놓으면 **컬럼을 자동 분석해 KPI 4개 + 차트 여러 개(시계열·순위 막대·구성비 도넛) + 상태 배지 표**가 한 화면에 생성 — **파싱·집계·렌더 전 과정이 브라우저 안(무전송)**. 매출 데이터가 서버로 나가지 않는다. 새로고침하면 사라지는 것이 의도(파일 흔적을 남기지 않는다).

## 실행

```bash
npm install
npm run dev / build / preview
npm run icons                       # 아이콘 재생성 (Pillow)
python3 scripts/generate-og.py      # OG 이미지 — 저장소 루트에서 실행
python3 scripts/generate-splash.py  # iOS 스플래시 17종 — 저장소 루트에서 실행
```

## 핵심 로직

- **csv.js — 인코딩·파싱·집계 정본**: 디코딩은 UTF-8로 먼저 읽고 깨진 문자(`�`) 개수가 0이 아니면 EUC-KR로 재디코딩(엑셀 "CSV로 저장" = CP949 대응) + BOM 제거. 구분자는 첫 줄에서 쉼표/탭/세미콜론 중 최다를 자동 선택. 파서는 따옴표·이스케이프(`""`)·필드 내 개행을 처리. 타입 추론은 컬럼 값의 80% 이상이 날짜면 `date`, 숫자면 `number`, 아니면 `category`(80% 규칙). `aggregate`는 카테고리축이면 값 내림차순 상위 10 + 나머지를 "그 외 N개"로 접고(시리즈를 늘리지 않는다), 날짜축이면 시간순 정렬 후 62건 초과 시 월(YYYY-MM) 단위로 리버킷.
- **자동 차트 생성 규칙 (`HomeView` roles)**: 컬럼 역할을 자동 배정한다. **전역 컨트롤**은 값(숫자 컬럼) 하나 + 집계(합계/평균/건수) 하나 — 모든 차트·KPI가 이 값 기준으로 그려진다. 기본 값 컬럼은 금액성 우선순위(`합계`>`매출·금액`>`가액·총액`>`단가`)로 고르고, 없으면 ID성(`순번·번호·id` 등)을 피해 첫 숫자 컬럼을 쓴다(순번을 합산하는 무의미 KPI 방지, 마스터 지적 2026-07-23). **필터 행**: 카테고리 컬럼(unique 2~30) 중 고유값 적은 순 최대 4개에 셀렉트(기본 "전체"), 필터는 `filteredRows`로 KPI·모든 차트·표에 공통 적용. **차트**(전부 `filteredRows` 기준): 날짜 컬럼이 있으면 시계열 라인(면적 채움), 카테고리 컬럼(unique 2~30) 각각에 가로 막대 최대 3개(고유값 적은 순), unique ≤ 6 카테고리 중 하나(필터에 안 쓴 것 우선)로 구성비 도넛 1개. `charts-grid`에 카드로 배치.
- **KPI 4개 + 상태 컬럼 감지**: ① 값 합계 ② 건수 ③ 값 평균 ④ **상태 KPI** — 값 집합이 `{완료, 대기, 취소, 미입금, 입금}` 부분집합인 카테고리 컬럼(`statusCol`)이 있으면 대기(`대기`·`미입금`) 건수·금액을 노란 테두리 타일로. 없으면 KPI 3개. 상태 컬럼은 표에서도 셀을 배지로 렌더(`완료·입금`=초록, `대기`=노랑, `취소·미입금`=빨강 — 색+텍스트 이중 인코딩).
- **차트 규약 (dataviz 스킬 적용·검증)**: 단일 시리즈 차트(라인·막대)는 **단일 색 `#4E8FD9`** — 정체성은 축 라벨이 담당, 범례 없음. 마크는 얇게 + 데이터 끝만 4px 라운드, 호버 툴팁 상시, 표 뷰 상시. 축은 하나(이중축 금지). SVG 텍스트는 텍스트 토큰 색을 `fill`로. **도넛만 시리즈 5색 고정 순서**(`#E4574B,#E8B93E,#4E8FD9,#57A867,#D96BA0`) — 이 팔레트는 다크 서피스(#1E2126)에서 `validate_palette.js` 결과 인접쌍 CVD 분리가 6.9 ΔE(6–8 floor band)로, **보조 인코딩이 있을 때만 합법**이다. 그래서 세그먼트 사이 **2px 간격**(배경 `#1E2126` base 링이 드러남)과 **범례(색점+라벨+%)**를 필수로 두어 색각 이상 사용자도 색만이 아니라 간격·라벨·%로 구분하게 한다. 6개 초과 시 상위 5+기타로 접는다(9번째 시리즈를 만들지 않는다).
- **레이아웃 (responsive)**: 이 앱만 앱 프레임을 넓힌다 — `.app` 기본 max-width 480px, `@media (min-width: 800px)`에서 960px + `.charts-grid` 2열 + KPI 1행. 모바일에선 차트 세로 스택, KPI 2×2, 필터 2열.
- **데이터 수명**: 파일 내용은 React 상태(`HomeView`의 `data`)에만 존재하고 localStorage·IndexedDB·서버 어디에도 저장하지 않는다. **새로고침 = 초기화가 의도** — 민감한 매출 데이터의 흔적을 남기지 않는다.
- **샘플 데이터 (`samples.js`)**: 파일 없이 즉시 시연하도록 **도메인 5종**을 내장 — `SAMPLES` 배열의 각 항목은 `{ id, name, emoji, desc, make }`이고 `make()`가 CSV 문자열(첫 줄 헤더)을 반환한다. ⓪ **거래 내역**(대표·기본, 첫 번째): 6개월 판매·렌탈, 순번/날짜/거래처/구분/품목/수량/단가/공급가액/부가세/합계/결제수단/입금상태 12컬럼, ~720건, 입금상태 완료 85%·대기 12%·취소 3% — 필터·상태 KPI·배지·도넛을 모두 보여주는 세라젬형 매출 대시보드 시연용. ① 카페 매출(주말 증가) ② 쇼핑몰 주문(주말·격주 프로모션 증가) ③ 고객센터 문의(월요일 최다) ④ 채용 지원. 전부 도메인별 고정 seed의 선형 합동 의사난수(Math.random 금지)로 **빌드마다 동일한 모양**을 생성(결정론적). 드롭존 아래 이모지 칩 그리드에서 선택 → `load()`.
- **PWA**: 앱 셸만 프리캐시 — 사용자 데이터는 애초에 로컬(상태)이라 오프라인 완결. 폰트는 런타임 캐시.

## 규약 (데모 시리즈 공통)

- UI·주석 한국어. 배경 `#1E2126`(주철)·텍스트 `#F2EFE9`(초크), 데이터 색은 단일 시리즈 `#4E8FD9`(도넛만 시리즈 5색). Black Han Sans + Noto Sans KR.
- 앱 아이콘: 막대 차트 3개(바닥 y=76 앵커, 가장 높은 오른쪽 막대만 데이터 색). 재생성 `npm run icons`.
- 모바일 퍼스트(maxWidth 480). 입력 필드 font-size 16px(iOS 자동 확대 방지).
- **시리즈 상호 링크 + 교육 카드**: `data/series.js` — 새 데모 앱 추가 시 모든 형제 앱 갱신·재배포(마스터 지시 2026-07-22).
- SEO·교육 홍보 규약: sitemap/robots·JSON-LD creator(잇다)·푸터 교육 카드 적용됨.
